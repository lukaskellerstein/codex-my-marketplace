# Content Patterns

Content writing patterns, formulas, and templates for website sections. Use these to write real, project-specific content rather than placeholder text.

---

## Headline Formulas

### Benefit-Driven
Focus on what the user gains.

```
"Get [benefit] without [pain point]"
"[Achieve outcome] without [sacrifice]"
"The easiest way to [desired result]"
"[Desired result], delivered [how]"
```

**Examples:**
- "Get enterprise security without enterprise complexity"
- "Launch faster without writing boilerplate"
- "The easiest way to manage your team's workflows"
- "Beautiful reports, delivered in seconds"

### Outcome-Focused
Quantify the result or set a timeframe.

```
"[Achieve X] in [timeframe]"
"From [before state] to [after state] in [timeframe]"
"[Number]x faster [activity]"
"[Activity] that takes [less time], not [more time]"
```

**Examples:**
- "Deploy to production in under 2 minutes"
- "From idea to live website in one afternoon"
- "10x faster code reviews with AI"
- "Onboarding that takes days, not months"

### Question Format
Engage the reader by addressing their desire.

```
"Ready to [desired outcome]?"
"What if [pain point] was not a problem?"
"Tired of [frustrating situation]?"
"What would you do with [freed resource]?"
```

**Examples:**
- "Ready to ship features your users actually want?"
- "What if managing invoices took 5 minutes, not 5 hours?"
- "Tired of context-switching between 12 different tools?"
- "What would you do with 10 extra hours every week?"

### Social Proof
Lead with credibility.

```
"Trusted by [number] [audience]"
"[Number] [audience] already [benefit]"
"The [category] tool rated #1 by [authority]"
"Join [number] [audience] who [action]"
```

**Examples:**
- "Trusted by 10,000+ engineering teams worldwide"
- "500,000 designers already create faster with Framekit"
- "The project management tool rated #1 on G2"
- "Join 25,000 founders who ship faster"

### Direct Value Proposition
State exactly what the product is and does.

```
"[Product name]: [one-line value prop]"
"[Category] for [specific audience]"
"The [adjective] [category] for [audience]"
"[Action verb] [what], [how/result]"
```

**Examples:**
- "Raycast: Your shortcut to everything"
- "Analytics for privacy-conscious teams"
- "The modern CRM for fast-moving sales teams"
- "Build APIs visually, deploy instantly"

## CTA Button Text Patterns

### Start / Begin (Primary CTAs)

| Button Text | When to Use |
|---|---|
| "Start Free Trial" | SaaS with free trial period |
| "Get Started Free" | Freemium products |
| "Try It Free" | Low-commitment trial |
| "Create Free Account" | When account creation is the conversion |
| "Start Building" | Developer tools, creative platforms |
| "Get Started" | Generic but effective — use when nothing specific fits |

### Action-Oriented (Engagement CTAs)

| Button Text | When to Use |
|---|---|
| "Book a Demo" | Enterprise, high-touch sales |
| "Schedule a Call" | Services, consulting, agencies |
| "See Pricing" | When pricing is on a separate page |
| "Request Access" | Beta products, waitlists |
| "Talk to Sales" | Enterprise with custom pricing |
| "Download Now" | Apps, resources, whitepapers |

### Value-Oriented (Urgency CTAs)

| Button Text | When to Use |
|---|---|
| "Save 50% Now" | Promotional pricing |
| "Unlock Premium" | Upgrade flow |
| "Claim Your Spot" | Limited availability |
| "Get Your Free [X]" | Lead magnets, free resources |
| "Join the Waitlist" | Pre-launch |
| "Reserve Early Bird Pricing" | Events, launches |

### Low Commitment (Secondary CTAs)

| Button Text | When to Use |
|---|---|
| "Learn More" | Generic secondary action |
| "See How It Works" | Below hero, drives to demo/video |
| "Watch Demo" | Video-based product demos |
| "View Examples" | Portfolios, case studies |
| "Explore Features" | Feature-rich products |
| "Read the Docs" | Developer tools |
| "Compare Plans" | Below pricing highlights |

### Button Text Anti-Patterns (Avoid These)

| Bad | Why | Better |
|---|---|---|
| "Submit" | Robotic, no value communicated | "Send Message" |
| "Click Here" | Meaningless, accessibility issue | "[Specific action]" |
| "Sign Up" | Too generic | "Start Free Trial" |
| "Buy" | Too blunt for most contexts | "Get [Product Name]" |
| "Enter" | Vague | "Join the Community" |
| "Go" | Lazy | "Search [Category]" |

## Section Copy Templates

### Hero Section — SaaS Product

**Version A (Benefit-driven):**
```
Headline: "Ship features 10x faster with AI-powered code review"
Subheadline: "Automated pull request analysis catches bugs, suggests improvements,
and approves clean code — so your team spends less time reviewing and more time building."
Primary CTA: "Start Free Trial"
Secondary CTA: "Watch 2-Min Demo"
```

**Version B (Problem-first):**
```
Headline: "Your team wastes 15 hours a week in code reviews"
Subheadline: "CodeBot automates the tedious parts — style checks, bug detection,
security scanning — and flags only what needs human judgment."
Primary CTA: "See It In Action"
Secondary CTA: "Read the Case Studies"
```

**Version C (Outcome-focused):**
```
Headline: "From pull request to production in minutes"
Subheadline: "CodeBot reviews, tests, and validates your code automatically.
Your team just approves and ships."
Primary CTA: "Start Building Faster"
Secondary CTA: "See How It Works"
```

### Features Section — Generic

**Section-level copy:**
```
Headline: "Everything you need to [achieve outcome]"
Subheadline: "Powerful features that work together seamlessly,
so you can focus on what matters — [core activity]."
```

**Feature card copy examples:**
```
Card 1:
  Title: "Real-Time Collaboration"
  Description: "Work together with your team in real time.
  See changes instantly, leave comments inline, and never
  overwrite each other's work."

Card 2:
  Title: "One-Click Deploy"
  Description: "Push to production with a single click.
  Automated testing, staging previews, and instant rollback
  if anything goes wrong."

Card 3:
  Title: "Built-In Analytics"
  Description: "Understand how users interact with your product.
  Track events, build funnels, and get alerts when metrics
  shift — no third-party tools needed."
```

### Testimonial Section

**Quote templates:**
```
Result-focused:
  "Since switching to [product], our [metric] improved by [X%].
  The team loves how [specific feature] saves them [time/effort]."
  — [Name], [Title] at [Company]

Problem-solution:
  "We tried [alternatives] but nothing handled [specific challenge].
  [Product] solved it in [timeframe] and the support team was incredible."
  — [Name], [Title] at [Company]

Emotional:
  "I finally feel in control of [area]. [Product] turned a chaotic
  [process] into something I actually enjoy managing."
  — [Name], [Title] at [Company]
```

### CTA Banner Section

**Version A (Urgency):**
```
Headline: "Ready to transform how your team ships code?"
Subheadline: "Join 5,000+ teams already shipping faster. Start free — no credit card required."
CTA: "Start Free Trial"
```

**Version B (Value-recap):**
```
Headline: "Faster reviews. Fewer bugs. Happier engineers."
Subheadline: "Start your free 14-day trial and see the difference in your first week."
CTA: "Get Started Free"
```

**Version C (Question):**
```
Headline: "What would your team build with 15 extra hours this week?"
Subheadline: "Let CodeBot handle code reviews while your engineers focus on features."
CTA: "Try CodeBot Free"
```

### FAQ Section

**Template answers should:**
- Start with a direct answer (yes/no, the key fact)
- Follow with 1-2 sentences of context
- End with a link or action if applicable

**Example Q&A pairs:**
```
Q: "Is there a free plan?"
A: "Yes. Our free plan includes up to 3 projects and 1,000 reviews per month —
   no credit card required. Most small teams never need to upgrade.
   See all plans →"

Q: "How long does setup take?"
A: "About 5 minutes. Connect your GitHub or GitLab repository, configure your
   review rules, and CodeBot starts reviewing your next pull request automatically."

Q: "Is my code secure?"
A: "Absolutely. We never store your source code. Analysis happens in ephemeral
   containers that are destroyed after each review. We are SOC 2 Type II certified
   and all data is encrypted in transit and at rest. Read our security page →"

Q: "Can I cancel anytime?"
A: "Yes, you can cancel your subscription at any time from your account settings.
   You will retain access to paid features until the end of your current billing period.
   No cancellation fees, no questions asked."
```

## Microcopy Patterns

Small text that makes a big difference in user experience.

### Form Labels and Placeholders

| Field | Label | Placeholder | Helper Text |
|---|---|---|---|
| Email | "Work email" | "you@company.com" | — |
| Password | "Password" | "At least 8 characters" | "Must include a number and uppercase letter" |
| Full name | "Full name" | "Jane Smith" | — |
| Company | "Company name" | "Acme Inc." | "Optional for solo users" |
| Message | "How can we help?" | "Tell us about your project..." | "We typically respond within 4 hours" |
| Phone | "Phone number" | "+1 (555) 000-0000" | "We'll only call if you ask us to" |
| URL | "Website" | "https://yoursite.com" | — |

### Error Messages

Write error messages that are specific, helpful, and human.

| Situation | Bad | Good |
|---|---|---|
| Invalid email | "Invalid input" | "That doesn't look like an email address. Check for typos?" |
| Required field | "Required" | "We need your email to send the report" |
| Password too short | "Error" | "Almost there — add 3 more characters to your password" |
| Server error | "500 Internal Server Error" | "Something went wrong on our end. We've been notified. Please try again in a few minutes." |
| No results | "No results found" | "No results for 'xyz'. Try a broader search or check your spelling." |
| Rate limit | "Too many requests" | "You're moving fast! Wait 30 seconds and try again." |

### Success Messages

| Situation | Message |
|---|---|
| Form submitted | "Message sent! We'll get back to you within 24 hours." |
| Account created | "Welcome aboard! Check your email to verify your account." |
| Payment complete | "You're all set! Your Pro plan is active immediately." |
| Password changed | "Password updated. You're all set." |
| File uploaded | "Upload complete. Your file is ready to use." |
| Newsletter subscribed | "You're in! Check your inbox for a welcome email." |

### Empty States

Empty states should explain what will appear, why it is empty, and what to do next.

```
No projects yet:
  Illustration: [folder icon or workspace illustration]
  Headline: "No projects yet"
  Description: "Create your first project to start organizing your work."
  CTA: "Create Project"

No search results:
  Illustration: [magnifying glass icon]
  Headline: "No results for '[query]'"
  Description: "Try adjusting your search or filters. You can also browse all items."
  CTA: "Clear Filters"

No notifications:
  Illustration: [bell icon]
  Headline: "All caught up"
  Description: "You have no new notifications. We'll let you know when something needs your attention."

No data yet (dashboard):
  Illustration: [chart icon]
  Headline: "Your analytics will appear here"
  Description: "Once your site receives traffic, you'll see visitor data, trends, and insights."
  CTA: "Install Tracking Code"
```

### Loading States

```
Initial page load: "[spinner] Loading your dashboard..."
Data fetching: "[skeleton placeholders]" (no text needed — use skeleton UI)
Long operation: "Generating your report... This usually takes about 30 seconds."
File upload: "Uploading 'design-v2.fig'... 67%"
AI processing: "Analyzing your code... Almost done."
```

### Tooltip Text

Keep tooltips under 15 words. They explain a UI element, not teach a concept.

```
Info icon next to "MRR": "Monthly Recurring Revenue — total of all active subscriptions"
Info icon next to "Churn rate": "Percentage of customers who cancelled this month"
Hover on disabled button: "Complete all required fields to continue"
Hover on beta badge: "This feature is in beta. Your feedback helps us improve it."
```
